FuncTree2 Documentation

FuncTree2 allows you to visualize and explore omics data in the hierarchical context of biological ontologies. By mapping omics data onto a pre-defined interactive treemap, based on the KEGG brite functional hierarchy, you can quickly and comprehensively understand the functional potential of your data, which will help you develop further hypothesis and scientific insights. Statistical tests, module pathway reconstruction, and different customization controls are built-in to help you in your quest.

Here you will learn about the different types of analysis available in FuncTree2 and the accepted data formats (Anchor). We also give you some usage tips and a sneak preview of what your output will look like.

  • KEGG Modules

Workflow

  • Module coverage: Module based mapping, where each module is given a coverage and abundance and then used for the pathway and brite abundance
  • Comparison: Two KO matrices, are used separatly for basinc mapping and the output is compared using Mann-Whitney U test
  • Display: Mapping without any processing. Lets the user take compltete control of generating his abundance data (Examnple)

Viewer

Ontology explorer

Summarized omics data are used to customize the Ontology explorer. Leaf values are summed or averaged to give the parent nodes a value, which is then reflected by a custom color.

  1. Zoom and pan
  2. Clicking on a node will collapse it.
  3. Tree could be rerooted
  4. search with autocomplete & highlight elements

Different coloring options

  • Layer: Automatically genrated colors for Ontology layers e.g. Brite L1, L2, Pathway, Module
  • Column: use the color code associated with a sample
  • Entry: Reflect user prederfined colors for each entry
